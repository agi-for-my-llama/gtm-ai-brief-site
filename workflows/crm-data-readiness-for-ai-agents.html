CRM Data Readiness For AI Agents

User

RevOps, SalesOps, or GTM systems owner.

Trigger

An AI agent is expected to read CRM records, recommend actions, update fields, or trigger downstream workflows.

Inputs

  • Target AI workflow.
  • Required CRM objects.
  • Required fields.
  • Field owners.
  • Allowed values.
  • Freshness rules.
  • Write permissions.

Output

  • Data readiness score.
  • Blocked fields.
  • Record exclusion rules.
  • Safe-read list.
  • Safe-write list.
  • Fix backlog.

Steps

  1. Pick one AI-assisted CRM workflow.
  2. List every object and field the agent needs.
  3. Check completeness, freshness, consistency, and ownership.
  4. Mark fields safe to read, safe to write, or blocked.
  5. Create exclusion rules for records that fail the readiness check.
  6. Log skipped records and required fixes.

Prompt

Evaluate this CRM workflow for AI agent readiness. Use the required objects, fields, field owners, allowed values, freshness rules, and write permissions. Return a readiness score, blocked fields, safe-read list, safe-write list, record exclusion rules, and a fix backlog.

Success Metric

No AI agent writes to CRM records unless required fields meet readiness rules.
