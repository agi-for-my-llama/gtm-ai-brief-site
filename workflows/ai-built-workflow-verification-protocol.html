AI-Built Workflow Verification Protocol

User

RevOps, Marketing Ops, SalesOps, or GTM engineer.

Trigger

AI drafts, edits, or configures a HubSpot, Salesforce, lifecycle, routing, or enrichment workflow.

Inputs

  • Natural-language workflow brief.
  • Generated workflow steps.
  • Source system.
  • Target objects.
  • Trigger rules.
  • Action rules.
  • Rollback plan.

Output

  • Verification checklist.
  • System-state checks.
  • Test records.
  • Approval decision.
  • Rollback steps.

Steps

  1. Convert the AI-generated workflow into explicit trigger, condition, and action rules.
  2. Check the live system state instead of trusting the AI summary.
  3. Run test records through each branch.
  4. Confirm exclusions and stop conditions.
  5. Review downstream field updates, notifications, and syncs.
  6. Approve, edit, or reject before activation.

Prompt

Create a verification protocol for this AI-generated GTM workflow. Translate the workflow into explicit trigger, condition, action, exclusion, and rollback checks. Define test records, expected outcomes, and the approval decision criteria.

Success Metric

Every AI-built workflow passes live system-state checks before touching production records.
