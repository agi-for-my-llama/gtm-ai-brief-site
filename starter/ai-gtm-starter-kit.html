AI GTM Starter Kit

Five workflows to test before buying another AI tool.

1. Answer Engine Visibility Check

Use when:

  • Buyers ask ChatGPT, Gemini, Perplexity, or Google AI answers for category recommendations.

Inputs:

  • Five high-intent buyer questions.
  • Your site.
  • Three competitor sites.
  • Current AI answers.

Output:

  • Missing questions.
  • Claim gaps.
  • Proof gaps.
  • Page updates.

Prompt:

Compare these AI answers, cited sources, competitor pages, and our page. Identify where we are missing buyer questions, proof, comparison language, or clear category framing. Recommend page edits we can make this week.

2. ChatGPT Ads Readiness Checklist

Use when:

  • A team wants to prepare for conversational ads without wasting early budget.

Checklist:

  • Define the buyer question that should trigger interest.
  • Write ad copy that fits decision research, not interruption.
  • Create a landing page that continues the same question.
  • Add conversion tracking.
  • Separate experimental budget from core search and paid social spend.

Test:

Build one campaign brief for a query where buyers compare options or ask what to do next.

3. Call Transcript To Deal Update

Use when:

  • Reps finish sales calls and CRM updates lag.

Inputs:

  • Transcript.
  • Current deal stage.
  • Previous notes.
  • Qualification fields.

Output:

  • Suggested stage.
  • Next step.
  • Follow-up email.
  • Risks.
  • Missing fields.

Prompt:

Use the transcript and deal history to draft a CRM update. Include only supported claims. Mark uncertain items as "needs rep confirmation."

4. Account Signal Triage

Use when:

  • Sales teams need a better daily account priority list.

Signals:

  • Hiring.
  • Funding.
  • Product launches.
  • New executive roles.
  • Website changes.
  • Technology adoption.

Output:

  • Priority score.
  • Reason.
  • Suggested opener.
  • Source link.

Prompt:

Score these accounts for outreach priority based on the supplied signals. Explain the buying trigger, likely pain, and best opener. Include source links.

5. Support Agent Deflection Map

Use when:

  • Support leaders want to deploy an AI agent without harming customer experience.

Inputs:

  • Last 100 tickets.
  • Help center articles.
  • Escalation rules.
  • Brand tone.

Output:

  • Safe-to-automate ticket types.
  • Human-required ticket types.
  • Missing knowledge base pages.
  • Pilot plan.

Prompt:

Cluster these support tickets by intent. Identify which categories are safe for AI draft or direct resolution, which require humans, and what documentation is missing.
